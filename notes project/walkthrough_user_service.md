## 2. Core Concepts Explained

### 2.1 JSON Web Token (JWT) Mechanics
A JWT is a compact, URL-safe string format used to securely transmit information between parties as a JSON object.

*   **Structure**:
    1.  **Header**: Defines the algorithm type (`HS256`) and format type (`JWT`).
    2.  **Payload (Claims)**: Contains session statements. We store:
        *   `Sub (Subject)`: User email.
        *   `role`: User authorization access levels.
        *   `exp`: Expiration time limit.
    3.  **Signature**: Generated by running HMAC SHA-256 on the base64-encoded Header and Payload combined with a private secret key. It prevents client-side data tampering.

```text
JWT Format: xxxxx.yyyyy.zzzzz (Header.Payload.Signature)
```

---

### 2.2 Security Hashing (BCrypt)
Storing passwords in plain text is a critical security vulnerability.
*   **The solution**: Hash passwords using **BCrypt**.
*   **How it works**: BCrypt uses a one-way mathematical function to transform a password into a 60-character scrambled code.
*   **Matches Check**: During login, `passwordEncoder.matches()` takes the password input, extracts the salt from the saved database hash, hashes the input using that salt, and compares the resulting hashes. **Passwords are never decrypted.**

---

### 2.3 Spring Security Configurations
The class `SecurityConfig.java` defines the framework rules:
1.  **Stateless Session**: Configured using `SessionCreationPolicy.STATELESS`. Spring Security will not allocate session storage memory, forcing clients to supply the authorization token on every HTTP request.
2.  **CSRF (Cross-Site Request Forgery)**: Disabled (`csrf.disable()`) because API client authentication parameters are passed in the HTTP headers (`Authorization: Bearer <JWT>`), which browsers do not automatically attach on cross-site form submissions.
3.  **Filter Chain Ordering**: We place our custom `JwtAuthenticationFilter` *before* the default `UsernamePasswordAuthenticationFilter`. This intercepts requests, parses the token, sets up the security principal, and enables downstream controller endpoints to process authenticated requests safely.

## 3. Endpoints & REST Architecture

*   `POST /api/users/register`: Collects user names, roles, emails, validates criteria, hashes passwords, and saves profile.
*   `POST /api/users/login`: Authenticates credentials and returns a secure JWT bearer token.
*   `GET /api/users/profile`: Retrieves profile details of the logged-in session. It queries the caller context through `SecurityContextHolder.getContext().getAuthentication().getPrincipal()` instead of accepting parameters, ensuring data encapsulation.
